# DBMS ERD Assignments

This repository contains a collection of ER (Entity-Relationship) diagrams designed for various real-world business scenarios as part of DBMS practice and coursework.

##  Objective

The goal of this repository is to practice and demonstrate strong database design fundamentals by modeling real-world systems with clean, normalized, and scalable schemas.

##  Concepts Covered

* Entity and Relationship identification
* One-to-Many and Many-to-Many relationships
* Use of junction tables
* Primary and Foreign Keys
* Normalization
* Real-world business modeling

##  Repository Structure

Each assignment is organized in a separate folder:

* `erd.png` → Visual representation of the ER diagram
* `schema.dbml` → Source code used to generate the diagram

##  Assignments Included

* Thrift & Handmade Store Database
* (More assignments will be added daily)

##  Tools Used

* eraser.io

##  Approach

Each ER diagram is designed by:

* Understanding the business problem
* Identifying key entities and relationships
* Structuring the schema to reflect real-world behavior
* Keeping the design clean and normalized

##  Note

This repository focuses only on database design (ER diagrams).
No frontend or backend implementation is included.
